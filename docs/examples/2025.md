# 4522 2025 competition examples

These examples come from [`4522_2025Competition`](https://github.com/TeamSCREAMRobotics/4522_2025Competition/tree/38f0984ae704c4e3da266547f38d9efcdccebe9b). That robot used the pre-26 package layout; current imports need the `com.teamscreamrobotics.` prefix.

## Model mechanism goals

The elevator extends `TalonFXSubsystem`, defines semantic goals, and translates `Length` into motor rotations. This keeps command code focused on `L2`, `L3`, or `BARGE` instead of raw rotations.

```java
public class Elevator extends TalonFXSubsystem {
  public Elevator(TalonFXSubsystemConfiguration config) {
    super(config, ElevatorGoal.FEED);
    resetPosition(0.0);
  }

  public enum ElevatorGoal implements TalonFXSubsystemGoal {
    FEED(Length.kZero),
    L2(Length.fromInches(24.5)),
    L3(Length.fromInches(40.5)),
    L4(Length.fromInches(64.8));

    public final Length height;
    private final DoubleSupplier targetRotations;

    ElevatorGoal(Length height) {
      this.height = height;
      this.targetRotations = () ->
          Conversions.linearDistanceToRotations(
              height, ElevatorConstants.PULLEY_CIRCUMFERENCE);
    }

    public ControlType controlType() {
      return ControlType.MOTION_MAGIC_POSITION;
    }

    public DoubleSupplier target() {
      return targetRotations;
    }
  }
}
```

## Describe field zones with typed lengths

The field constants combine `Length`, `HexagonalPoseArea`, and `RectangularPoseArea` to make containment checks readable:

```java
public static final RectangularPoseArea FIELD_AREA =
    new RectangularPoseArea(Translation2d.kZero, FIELD_DIMENSIONS);

public static final Length CORAL_DIAMETER = Length.fromInches(4.5);

public static final HexagonalPoseArea BLUE_REEF =
    new HexagonalPoseArea(
        BLUE_REEF_CENTER,
        Length.fromMeters(10),
        Rotation2d.fromDegrees(-30));
```

## React once when a selection changes

`RunnableUtil.RunOnce` avoids rebuilding the autonomous command on every robot loop:

```java
private final RunOnce autoCommandReloader = new RunOnce();

public void robotPeriodic() {
  autoCommandReloader.runOnceWhenChanged(
      () -> autonomousCommand = robotContainer.getAutonomousCommand(),
      robotContainer.getAutoSelector().getSelected());
}
```

[See the 2025 robot loop](https://github.com/TeamSCREAMRobotics/4522_2025Competition/blob/38f0984ae704c4e3da266547f38d9efcdccebe9b/src/main/java/frc2025/Robot.java#L30-L76).

## Flip field values by alliance

The robot selects a blue or red pose at the point of use:

```java
Pose2d target = AllianceFlipUtil.get(
    FieldConstants.BLUE_BARGE_ALIGN,
    FieldConstants.RED_BARGE_ALIGN);
```

Use `AllianceFlipUtil.get(...)` when both alliance values are known. Use its pose/translation flip methods when one blue-origin value should be reflected across the field.
