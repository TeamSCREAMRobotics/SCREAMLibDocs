# 4522 2026 competition examples

These examples come from [`4522_2026Competition`](https://github.com/TeamSCREAMRobotics/4522_2026Competition/tree/e9d3ad1471c68ffa779655b75c4d56b9b7730325) and use SCREAMLib's current `com.teamscreamrobotics.*` packages.

## Configure a TalonFX mechanism

The hood constants assemble the reusable motor configuration, PID slot, limits, and Motion Magic values before the subsystem is constructed:

```java
public static final TalonFXSubsystemConfiguration HOOD_CONFIG =
    new TalonFXSubsystemConfiguration();

static {
  HOOD_CONFIG.name = "Hood";
  HOOD_CONFIG.masterConstants =
      new TalonFXConstants(
          new CANDevice(8),
          InvertedValue.Clockwise_Positive);

  HOOD_CONFIG.slot0 =
      new ScreamPIDConstants(90.0, 0.0, 0.0)
          .getSlot0Configs(new FeedforwardConstants(0.0, 0.0, 0.0, 0.0));

  HOOD_CONFIG.neutralMode = NeutralModeValue.Brake;
  HOOD_CONFIG.sensorToMechRatio = 30.8333;
  HOOD_CONFIG.enableSupplyCurrentLimit = true;
  HOOD_CONFIG.supplyCurrentLimit = 15.0;
  HOOD_CONFIG.maxUnitsLimit = 0.15;
  HOOD_CONFIG.acceleration = 20.0;
  HOOD_CONFIG.cruiseVelocity = 30.0;
}
```

[See `HoodConstants`](https://github.com/TeamSCREAMRobotics/4522_2026Competition/blob/e9d3ad1471c68ffa779655b75c4d56b9b7730325/src/main/java/frc2026/tars/subsystems/shooter/hood/HoodConstants.java#L20-L48).

## Configure master and follower motors

The flywheel uses the same configuration with three followers and different inversion per device:

```java
FLYWHEEL_CONFIG.masterConstants =
    new TalonFXConstants(new CANDevice(0), InvertedValue.CounterClockwise_Positive);
FLYWHEEL_CONFIG.slaveConstants = new TalonFXConstants[] {
    new TalonFXConstants(new CANDevice(1), InvertedValue.CounterClockwise_Positive),
    new TalonFXConstants(new CANDevice(2), InvertedValue.Clockwise_Positive),
    new TalonFXConstants(new CANDevice(3), InvertedValue.Clockwise_Positive)
};
```

[See `FlywheelConstants`](https://github.com/TeamSCREAMRobotics/4522_2026Competition/blob/e9d3ad1471c68ffa779655b75c4d56b9b7730325/src/main/java/frc2026/tars/subsystems/shooter/flywheel/FlywheelConstants.java#L20-L55).

## Aim with geometry and typed distance

The shooter uses `Length`, `ScreamMath`, and `AllianceFlipUtil` together:

```java
public Length getShotDistance(Translation2d target) {
  return Length.fromMeters(
      getFieldToShooter().getTranslation().getDistance(target));
}

robotState.setDrivetrainTarget(
    ScreamMath.calculateAngleToPoint(robotPose.getTranslation(), target)
        .plus(Rotation2d.k180deg));

Translation2d hub = AllianceFlipUtil.get(
    FieldConstants.Hub.hubCenter,
    FieldConstants.Hub.oppHubCenter);
```

[See the 2026 shooter](https://github.com/TeamSCREAMRobotics/4522_2026Competition/blob/e9d3ad1471c68ffa779655b75c4d56b9b7730325/src/main/java/frc2026/tars/subsystems/shooter/Shooter.java#L115-L207).

## Feed Limelight estimates into odometry

The vision manager updates robot orientation, chooses MegaTag 1 while disabled and MegaTag 2 while enabled, rejects poor estimates, and passes standard deviations into drivetrain odometry:

```java
LimelightHelpers.SetRobotOrientation(
    limelight.name(),
    drivetrain.getHeading().getDegrees(),
    drivetrain.getYawRate().getDegrees(),
    0, 0, 0, 0);

PoseEstimate estimate = DriverStation.isDisabled()
    ? LimelightHelpers.getBotPoseEstimate_wpiBlue(limelight.name())
    : LimelightHelpers.getBotPoseEstimate_wpiBlue_MegaTag2(limelight.name());
```

[See `VisionManager`](https://github.com/TeamSCREAMRobotics/4522_2026Competition/blob/e9d3ad1471c68ffa779655b75c4d56b9b7730325/src/main/java/frc2026/tars/subsystems/vision/VisionManager.java#L134-L173).
